Invert A Binary Tree

WIP

This is a Rosetta Code post.

I’m fortunate to work with super smart people, sometimes these smart people tell me about code problems - today I was asked if I knew how to invert a binary tree. (I did not know how)

Story

An inverted form of a Binary Tree is another Binary Tree with left and right children of all non-leaf nodes interchanged. You may also call it the mirror of the input tree.

  • https://afteracademy.com/blog/invert-a-binary-tree

Some notes from my google-neering

  • A Tree is used to represent data in a hierarchical format

  • Every node in a tree has 2 components (Data and References)

  • The top node of the tree is called the Root node and the 2 products under it are called “Left Subtree” and “Right Subtree”.

  • There are 3 types of binary trees:

    • Strict Each node has either two children or none.
    • Full Each Non-leaf node has two children and all the leaf nodes are at the same level.
    • Complete If all the levels are completely filled, except the last level and the last level has all the keys as left as possible

  • https://www.c-sharpcorner.com/article/tree-data-structure/

Task

Given the binary tree from the story above, invert it so its values show as the example Output. Its important to remember that its a mirror image so the left and right children are swapped. (Dont try flip it so the root node is at the bottom)
