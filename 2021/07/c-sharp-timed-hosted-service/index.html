Story

Consider the following events feed for a shopping cart below. These events are CRUD actions CART_CREATE, CART_UPDATE and CART_DELETE. The following query string parameters are used:

  • watermark=42 this means where events.id >= 42
  • page=3 this means pagination page 3
  • size=25 this means the page size is 25, ie: the max events count will be 25
Verb URI example
GET http://localhost:3331/events?watermark=42&page=3&size=25

Task

Create a hosted service that fulfills the following acceptance criteria:

  • a watermark needs to be used so events are not processed again
  • use page and size parameters
  • the worker should run infinitely until the process is interupted with ctrl+c
  • consumes this feed and persists the CRUD event to a database as a CREATE, UPDATE or DELETE

Solution

This is the simplest no frills solution, from here it could be adapted to inlude message queues and better separation of concerns.
