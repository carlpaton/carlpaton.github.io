Data Structures and Algorithms in C#

Data structures and Algorithms apply to any programining language, the examples I seek to collate are in C# as its currently my language of choice.

Why care?

Suitable data structures affect the performance of your application. You can increase quality and performance by choosing the right data structure and accomanying algorithm. This feels like something we should care about. Additionally the topics below are worth understanding.

Big O notation

  • Linear Time vs. Logarithmic Time — Big O Notation
  • https://en.wikipedia.org/wiki/Big_O_notation

“In computer science, it is useful in the analysis of algorithms” - wikipedia.org

  • Logarithmic time (Complexity Analysis) is faster than a Linear time operations because it runs in O(log n)
  • One of my peers once explained this to me as being able to use this in regards to problem solving: Example: what’s the big O of xs.map(...).filter(...).find(...)

Sliding Window Technique

Involves manipulating two pointers or two indices at the same time as you’re traversing thought something like a string or an array. Basically you have a left pointer/index and a right pointer/index and your move them simultaneously in an array or a string. (could be to count frequency of characters)

Recursion

A lot of problems can be solved easily with recursion (as apposed to iteratively). This produces cleaner and less code than an iterative approach. Example problems that help you understand recursion include:

  • n fibonacci
  • calculate height of binary tree or the depths of nodes in a binary tree

Dynamic Programming (DP)

It means being able to solve a problem that is complex by first solving a smaller version of that problem. To solve that small version of the problem you solve an even smaller version of that problem, until you can solve a trivial version of the problem. The only way to get good at DP is to do a lot of DP!

Binary Search Tree

  • Tutorial 19 - Binary Search Trees in C#
  • Binary Search Tree implemented in C#

Also see Invert A Binary Tree

  • None-linear, meaning they have a Left and a Right pointer
  • Searching
    • if the value of the data is < this node, go left
    • if the value of the data is >= this node, go right
  • Each pass cuts the data in half
  • Logarithmic performance for
    • Searching
    • Inserting new data
    • Updating data
  • Dynamic
  • An un-balanced tree (possibly from sorted data) will have performance issues

Example log base 2 of 16 = 4 this is how many times we can chop 16 in half.

Merge Sort

  • https://www.geeksforgeeks.org/merge-sort/

Merge Sort is a Divide and Conquer algorithm. It divides the input array into two halves, calls itself for the two halves, and then it merges the two sorted halves.
